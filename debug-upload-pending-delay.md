[OPEN] upload-pending-delay

# 调试目标
- 现象：点击发送录音后，`/v1/process-upload` 长时间 pending，最终才返回。
- 目标：定位主要耗时落在哪一段，并给出最小修复方案。

# 当前假设
1. 首次请求触发 `wav2vec2` 或 `faster-whisper` 模型下载/初始化，导致长时间等待。
2. 双通道 ASR 推理耗时过高，且当前运行环境没有做预热，导致首轮请求明显慢。
3. Ark `responses` 接口耗时较高，但不是唯一瓶颈。
4. 浏览器上传的音频过长或编码处理较慢，导致进入推理前已消耗大量时间。

# 证据计划
- 查看当前服务运行日志，确认 pending 期间是否有模型下载、模型加载或推理相关输出。
- 用接口直接发一个极短音频请求，记录整体耗时。
- 分段记录：上传落盘、双通道 ASR、融合、Ark 请求各阶段耗时。

# 当前状态
- 待收集运行时证据，尚未修改业务逻辑。
